UI As-Is / To-Be Report

This report compares the current widget field-selection UI against the proposed nested JSON field-selection enhancement.

Current status: no UI code changed yet in this session.

As-Is

Current UI

  • The field selector already exists for table and json_preview widgets after Test Fetch.
  • It shows only top-level fields from the JSON response, such as name, language, and detail.
  • Nested values like detail.obj.key or detail.name cannot be selected directly.
  • Table rendering reads flat keys only, so nested values are not shown as separate columns.
  • JSON Preview can filter only by top-level selected fields.
To-Be

Proposed UI

  • The same field selector UI remains in place; no major layout redesign is proposed.
  • After Test Fetch, the selector will list dotted nested paths like detail.obj.key and detail.name.
  • Visitors can choose deep object properties without changing backend persistence.
  • Table widget headers will show the selected dotted paths and render their resolved values.
  • JSON Preview will rebuild a reduced nested object containing only the selected branches.

Example Response

For this response:

{ "name": "Adeel Solangi", "language": "Sindhi", "detail": { "obj": { "key": "value" }, "name": "test" } }

Area As-Is To-Be
Selectable fields name, language, detail name, language, detail, detail.obj, detail.obj.key, detail.name
Table widget Cannot show value and test as independent selected nested columns. Can show columns detail.obj.key = value and detail.name = test.
JSON Preview widget Can keep only top-level detail, but not trim it to specific child fields. Can render only the selected nested branch, for example { "detail": { "obj": { "key": "value" }, "name": "test" } }.
